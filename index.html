K-Means

Watch unlabeled points become structure.

K-means is a fast, visual way to partition observations into compact groups. This experience stays focused on the decisions that change the answer: choosing K, inspecting within-cluster variation, handling scale, and spotting when a result is only a local optimum.

Unsupervised Learning

No labels. Only proximity.

Unsupervised learning starts without a known outcome column. The algorithm only sees the features and asks: which observations naturally sit near one another? K-means answers that question by building groups around centroids.

What to look at

Before color is added, the points are only geometry. The cluster labels are created by the method, not discovered from a hidden answer key.

Decision to make next

Decide whether distance is meaningful in your variables. If one variable is on a much larger scale, K-means will treat it as more important.

Try this in R 
x_scaled <- scale(df[, vars])
fit <- kmeans(x_scaled, centers = 3, nstart = 25)

Algorithm Loop

The algorithm is a repeated negotiation.

K-means alternates between two operations: compute centroids from the current group assignments, then reassign every point to the nearest centroid. The loop stops when no observation wants to move.

3
  1. Choose K
  2. Random start
  3. Compute centroids
  4. Reassign points
  5. Repeat until stable
What to look at

The interesting moments are when points jump across a boundary and when a centroid moves enough to drag the next boundary with it.

Decision to make next

If the loop stabilizes quickly but the clusters still look implausible, try new random starts before changing the whole modeling strategy.

Choosing K

Read the bend, not just the number.

The K slider updates the clustering instantly. Pair the scatterplot with the elbow curve and silhouette signal: you want a K that cuts within-cluster variation sharply without over-fragmenting the structure.

3
What to look at

Look for the point where the elbow starts to flatten. Then check whether the scatterplot still tells a coherent story at that same K.

Decision to make next

If the elbow is ambiguous, inspect silhouette and cluster sizes. Tiny, unstable clusters are usually a sign that K is too large.

Try this in R 
wss <- sapply(1:8, function(k) {
  kmeans(x_scaled, centers = k, nstart = 25)$tot.withinss
})

Random Starts

Good geometry can still begin badly.

Because K-means starts from a random configuration, two runs on the same data can end at different solutions. The right visual question is not “did it converge?” but “did it converge to a good minimum?”

What to look at

Compare the left and right panels. The same points can be partitioned very differently depending on where the centroids start.

Decision to make next

Increase the number of random starts until the best solution becomes stable. Treat a single start as a draft, not a final answer.

Try this in R 
fit_1 <- kmeans(x_scaled, centers = 3, nstart = 1)
fit_25 <- kmeans(x_scaled, centers = 3, nstart = 25)

Scaling

Raw units decide the answer unless you stop them.

In this example, sock counts span a much wider range than computer counts. Without scaling, K-means largely clusters by socks. With scaling, both variables get a fair vote and the groups reorganize around a more meaningful business pattern.

What to look at

Watch the color pattern rotate when scaling is turned on. The point locations do not change, only the distance calculation does.

Decision to make next

If one feature uses larger numbers for purely unit-related reasons, standardize before you judge cluster quality.

Try this in R 
x_raw <- df[, c("socks", "computers")]
x_scaled <- scale(x_raw)
kmeans(x_scaled, centers = 2, nstart = 25)

Outliers

One stray point can bend a centroid.

K-means always assigns every observation to a cluster. That means an outlier is not ignored. It gets absorbed somewhere, often pulling the nearest centroid away from the dense part of its own cluster.

What to look at

The dashed centroid marks the clean solution. When the outlier appears, the live centroid shifts toward it and the boundary changes with it.

Decision to make next

Investigate extreme observations before trusting the final partition. The right fix could be transformation, scaling, or a different clustering method.

Try this in R 
fit <- kmeans(scale(df[, vars]), centers = 2, nstart = 25)
df$cluster <- fit$cluster

Interpretation

Final clusters are summaries, not truth.

A final K-means result should be interpreted by comparing cluster centers, spread, and size. Click a cluster card to isolate it and read its centroid in the same language as your variables.

What to look at

Separation matters, but so does meaning. A cluster is useful when its center and spread translate into a recognizable pattern in the real problem.

Decision to make next

Summarize the original variables by cluster. If the profiles are not interpretable, revisit preprocessing or the chosen K.

Try this in R 
df$cluster <- fit$cluster
aggregate(df[, vars], list(cluster = df$cluster), mean)

From Demo To Real Data

Use the same sequence on an actual dataset.

  • Start with variables where Euclidean distance makes domain sense.
  • Scale first unless raw units are intentionally meaningful.
  • Inspect several values of K with elbow, silhouette, and cluster size.
  • Run many random starts and keep the best objective value.
  • Check for outliers before you interpret centroids as real profiles.
  • Translate the final labels back to the original variables for interpretation.

Hierarchical

Grow the tree before you cut it.

Hierarchical clustering builds nested relationships from the bottom up. Instead of committing to a cluster count at the start, you watch observations fuse, compare how “close” is defined, then cut the dendrogram where the structure becomes convincing.

What It Solves

See every possible grouping in one structure.

Hierarchical clustering is useful when you do not want to lock in a cluster count upfront. The dendrogram keeps the nested story intact, so you can inspect broad groups first and then zoom into finer splits only if they add meaning.

What to look at

A dendrogram stores both local joins and higher-level structure. You are reading a sequence of fusions, not a final partition.

Decision to make next

Decide whether you care more about nested interpretation or a single flat answer. If the tree matters, hierarchical clustering is often the better explainer.

Try this in R 
hc <- hclust(dist(scale(df[, vars])), method = "average")
plot(hc)

Reading The Tree

Leaves stay fixed. Heights tell the story.

Use the slider to move through the first fusions. Every step joins the two closest clusters under the chosen linkage rule. The only distance that matters on the dendrogram is vertical fusion height.

3
What to look at

Horizontal position is just ordering. The height of each merge is the actual dissimilarity. High late merges suggest strongly separated groups.

Decision to make next

If early merges look sensible and late merges jump sharply, the data likely contains a few meaningful groups rather than a smooth continuum.

Try this in R 
hc <- hclust(dist(scale(df[, vars])), method = "complete")
plot(hc, hang = -1)

Cut Height

Pick the partition after you see the tree.

Move the cut line upward and downward. High cuts produce a few broad clusters. Low cuts create many fine-grained groups. The right cut is the one that preserves the separation you can defend.

55%
What to look at

Large empty vertical gaps suggest natural cut regions. A cut through a tall gap usually preserves meaningful separation.

Decision to make next

Compare the cluster count from your chosen cut to the actual scatterplot. Keep the cut only if the groups are still interpretable in the original space.

hc <- hclust(dist(scale(df[, vars])), method = "average")

Linkage

The merging rule changes the tree shape.

Linkage answers a hidden question: when are two multi-point clusters close enough to fuse? Compare complete, single, average, and centroid linkage on the same geometry.

What to look at

Single linkage is prone to chaining. Complete linkage stays conservative. Average often balances the two. Centroid can create surprising merge order reversals.

Decision to make next

Pick the linkage whose implied cluster shape matches the science of the problem, then verify that the result is stable under small modeling changes.

Try this in R 
hc_complete <- hclust(dist(x), method = "complete")
hc_single <- hclust(dist(x), method = "single")

Distance Measure

Magnitude and shape are not the same thing.

Here the observations are short profiles rather than scatter points. Euclidean distance groups by raw magnitude. Correlation distance groups by pattern shape, regardless of overall level.

What to look at

Under correlation distance, parallel profile shapes move together even if one customer buys much more than another.

Decision to make next

Ask whether absolute size or relative pattern matters more in your domain. The answer should drive the distance metric before you ever draw the tree.

Try this in R 
d_euclid <- dist(x)
d_corr <- as.dist(1 - cor(t(x)))
hc_corr <- hclust(d_corr, method = "average")

Scaling

The tree changes when units change.

Hierarchical clustering is just as sensitive to scaling as K-means when Euclidean distance is used. Toggle raw units and scaled values to see how the dendrogram and the recovered groups change together.

What to look at

When scaling is off, the widest-range variable controls the earliest merges. The tree itself shifts because the distance matrix shifts.

Decision to make next

If the unit system is arbitrary, standardize before interpreting branch heights as meaningful dissimilarities.

x_raw <- df[, c("socks", "computers")]
hc_scaled <- hclust(dist(scale(x_raw)), method = "average")

Outliers

Outliers appear as lonely, expensive branches.

In a dendrogram, an outlier often stays alone until a very high merge. That can be useful because the tree makes the separation visible, but it can also dominate the final cut decision if you are not careful.

What to look at

The outlier’s branch stays isolated until a tall final merge. That height is the clue that the point does not fit the main structure.

Decision to make next

Decide whether the outlier is noise, a rare but valid subgroup, or a signal to use a different distance measure.

Try this in R 
hc <- hclust(dist(scale(df[, vars])), method = "complete")
plot(hc)
rect.hclust(hc, k = 2)

Interpretation

Read the final cut as a nested story.

Use the final cut slider to decide how much detail you want. Each cluster card summarizes the members captured at that height, so you can compare broader families with finer subgroups using the exact same tree.

58%
What to look at

Final interpretation comes from the chosen cut and the branch heights above it. A cluster that fuses late is genuinely more distinct.

Decision to make next

Translate the members in each branch back to the original variables or entities. The tree is only useful if the clusters tell a recognizable domain story.

Try this in R 
groups <- cutree(hc, h = 3.2)
aggregate(df[, vars], list(cluster = groups), mean)

From Demo To Real Data

Carry the same decisions into practice.

  • Decide whether magnitude or shape is the right notion of similarity.
  • Scale variables before Euclidean distance unless raw units are intentional.
  • Compare linkage rules and keep the one that matches the expected cluster shape.
  • Inspect the dendrogram for large height gaps before choosing a cut.
  • Check whether outliers are forming singleton branches that dominate the tree.
  • Summarize original variables after cutting so the branches become interpretable.

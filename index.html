K-Means

Watch dots form groups.

Dots that sit near each other can become one group.

K-means keeps moving each dot to the nearest group middle until the picture stops changing. That group middle is called a centroid.

watch dots move pick the group count try a new start open the R step

Start

Start with dots and no labels.

At first, the dots are just dots.

The computer only looks at which dots are close. Then it makes groups and finds a group middle for each one. That middle is called a centroid.

What to notice

First you see only gray dots. After one click, the colors show the groups the computer made.

Try this in R 
x_scaled <- scale(df[, vars])
fit <- kmeans(x_scaled, centers = 3, nstart = 25)

How it works

Watch the dots move to the nearest group.

Each dot goes to the closest group.

First you pick how many groups you want. Then the computer makes a first guess, finds each group middle, and moves the dots again until nothing changes.

First guess

A bad first guess can give a bad grouping.

If the first guess is bad, the answer can be bad too.

K-means begins from random starting spots. Run it many times and keep the best answer, not just the first one.

What to notice

The dots never move between these two pictures. Only the first guess changed.

Scaling

Big numbers can take over.

One axis can win just because its numbers are bigger.

That can hide the real pattern. Scaling makes the number sizes more equal so both axes get a fair say.

What to notice

The dots stay still. Only the way we count closeness changes.

Outliers

A weird point can pull the group middle.

One far dot can tug the group away.

Every dot must join some group, so one lonely point can drag the group middle, called the centroid, away from the main pile.

What to notice

Interpretation

Read what each group is like.

Now ask what kind of dots are inside each group.

A good grouping is not only separated. Each group should also mean something clear when you describe it in ordinary words.

What to notice

Separation is not enough. You also want a group you can explain in a simple sentence.

Hierarchical

Build the group tree.

Dots join into bigger and bigger groups.

This picture is a group tree, called a dendrogram. You watch dots join first, then you choose where to cut the tree.

watch joins move the cut change the rule open the R step
Group tree growing live

Start

Start with one tree, not one answer.

This method keeps many possible group answers in one tree.

You do not choose the final number of groups at the start. The tree keeps every join so you can decide later.

What to notice

This picture is not one final answer. It is one tree that can give you many different group answers.

Try this in R 
hc <- hclust(dist(scale(df[, vars])), method = "average")
plot(hc)

How to read it

The higher the join, the less similar the groups.

The closest dots join first.

Read the tree by height. A high join means the two parts were less alike, or more dissimilar.

Join rule

Change the rule for joining groups.

Different join rules can build different trees.

A join rule decides how we measure closeness between whole groups, not just between single dots. That choice can change the tree a lot.

What to notice

Some rules keep groups tight. Some rules let groups join into long chains.

What counts as close?

Choose what “close” means.

Things can be close in size or close in shape.

One choice cares about how much there is. The other cares about whether the ups-and-downs match. Pick the one that fits your question.

What to notice

Two lines can have the same shape even when one sits much higher than the other.

Try this in R 
d_euclid <- dist(x)
d_corr <- as.dist(1 - cor(t(x)))
hc_corr <- hclust(d_corr, method = "average")

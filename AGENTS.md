# Clustering site rules

## Product goal
This site must teach clustering in a child-simple, visually intuitive way while keeping a premium visual style.

## Layout rule
All major sections must use:
- text on the left
- visual on the right

Do not alternate the layout.

## Teaching rule
Always explain the idea before naming the technical term.
Use one child-simple first sentence in every section.

## Style rule
Avoid abstract or slogan-like wording.
Prefer literal teaching language.
Keep the site clean, centered, and not dashboard-like.

## Visual rule
Visuals must do most of the teaching.
Use animation, direct labels, and bigger graphics.
Controls must stay close to the visual they affect.

## UX rule
Users must always know where to start.
Keep a clear “Start here” pattern in each tab.

## R code rule
Use short, contextual “Try this in R” snippets.
Do not dump large code blocks unless needed.
